0. 通用复用规则见 `skill.txt`（软链接到共享的 `../myutils/generalskill.txt`）；本文件只保留 `codex_docs` 项目特有要求。
1. 项目目的：同步并维护 Codex CLI 相关官方在线文档的本地 Markdown 镜像，在設置或使用codex時可以參考這些文檔
2. 官网与仓库文档统一存放在 `docs/` 下，并按来源分命名空间保存：OpenAI Developers 文档落在 `docs/developers/...`，`openai/codex` 仓库文档落在 `docs/github/...`；抓取来源统一记录在根目录 `urls/`，也按镜像结构保存。
3. `sync_all_docs.py` 负责全量发现、补齐、刷新，并会清理不再属于当前 Codex cli 文档集合的 stale `docs/` / `urls/` 镜像文件。
4. 发现本地 `.md` 为空、正文极少或明显截断时，优先视为抓取不完整，先重新下载，再使用内容；但对 `openai/codex` 仓库里那类“短跳转 stub 文档”，如果其内容明确指向 Developers 正式页面，不要误判为坏文档。
5. 不见得需要精读每一个文档, 阅读 Codex cli 文档时可以使用程序，先看标题层级，再看超链接，最后只对当前任务相关页面做深入阅读。
6. 如果本地 Markdown 链接值得读时：本地已有就直接读；本地没有就先下载下来，再继续阅读或分析。
